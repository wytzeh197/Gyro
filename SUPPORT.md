# Support

Gyro is alpha software and does not yet have a guaranteed support service.
Community reports are still valuable and help make the first release safer.

## Get Help

- Check the [README](README.md) for requirements and source-build instructions.
- Run `pnpm doctor` for repository development setup.
- Run `cargo run -p gyro-cli -- doctor` for CLI, storage, Git, app IPC, and
  provider-readiness checks.
- Search [existing issues](https://github.com/wytzeh197/Gyro/issues) before
  opening a new one.
- Open a [bug report](https://github.com/wytzeh197/Gyro/issues/new?template=bug_report.yml)
  for reproducible defects or a
  [feature request](https://github.com/wytzeh197/Gyro/issues/new?template=feature_request.yml)
  for product proposals.

When reporting a problem, include the Gyro version or commit, macOS version,
whether you used the app or CLI, reproduction steps, and redacted diagnostic
output. Never post API keys, session databases, private source code, complete
provider transcripts, or other secrets.

Security vulnerabilities must not be filed as public issues. Follow the
[private reporting instructions](SECURITY.md#report-a-vulnerability).
