# Gyro v0.1.0-alpha.26.1

Gyro Alpha 26.1 makes provider connections more resilient to transient health
check failures and brings a more consistent visual hierarchy to Settings.

## Download and install

Gyro requires macOS 14+ and is not Apple-signed or notarized. The application
has a complete ad-hoc signature, so macOS can verify the bundle structure, but
the first launch still requires Apple's one-time **Open Anyway** approval.

Use the [Gyro download site](https://usegyro.io/) to choose a
build, then follow the
[macOS installation guide](https://github.com/wytzeh197/Gyro/blob/main/docs/install-macos.md).
Do not disable Gatekeeper or remove quarantine globally.

- [Apple Silicon DMG](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.26.1/Gyro_0.1.0-alpha.26.1_aarch64.dmg)
- [Intel DMG](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.26.1/Gyro_0.1.0-alpha.26.1_x64.dmg)
- [SHA256SUMS](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.26.1/SHA256SUMS)

## What changed

- Provider health probes no longer persist transient failures as sign-outs or
  disable providers while checking their status.
- Successful probes still restore a provider to connected state, while
  definitive missing-login and missing-installation states continue to block
  execution clearly.
- Non-zero and temporarily unavailable provider commands now surface as
  warnings instead of ambiguous health results.
- Settings pages use a consistent page and group heading hierarchy, with
  refined hairline borders and focus states.
- Settings title-bar spacing now holds at Gyro's minimum supported window size,
  and theme controls expose their selected state to assistive technology.
- Regression checks cover provider connection retention, runtime health
  mapping, settings semantics, and minimum-window layout expectations.

## Behavior changes and known limits

- A transient provider warning can leave an already connected provider
  available. Explicit not-installed and not-logged-in results still require
  corrective setup before launch.
- Gyro is still an unsigned public Alpha. A warning-free first launch requires
  Apple Developer ID signing and notarization, which this release does not use.
- Managed Macs may prohibit **Open Anyway**. Gyro does not bypass device policy.
- Apple Silicon and Intel remain separate downloads; there is no universal DMG.
- Homebrew installs the `gyro` CLI only. Gyro.app remains a direct DMG install.
- Alpha 26 sessions and settings remain compatible; no database or session
  schema migration is required.

## Upgrade and rollback

Alpha 26 sessions, attachments, plans, approvals, projects, and local provider
state remain compatible with Alpha 26.1. Update from Gyro or replace the
existing app with the matching Alpha 26.1 DMG.

To roll back, quit Gyro, back up
`~/Library/Application Support/Gyro/`, and reinstall
[`v0.1.0-alpha.26`](https://github.com/wytzeh197/Gyro/releases/tag/v0.1.0-alpha.26).

## CLI

This release does not change CLI command behavior. After the release Formula
reaches the tap:

```bash
brew tap wytzeh197/tap
brew trust --formula wytzeh197/tap/gyro
brew install gyro
```

Direct archives remain available for
[Apple Silicon](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.26.1/gyro-cli-0.1.0-alpha.26.1-aarch64-apple-darwin.tar.gz)
and
[Intel](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.26.1/gyro-cli-0.1.0-alpha.26.1-x86_64-apple-darwin.tar.gz).

## Alpha notice

This is a GitHub-built public macOS Alpha for Apple Silicon and Intel. Updater
archives are signed with Gyro's updater key, while app bundles use ad-hoc code
signing and DMGs are not Apple-signed or notarized.
