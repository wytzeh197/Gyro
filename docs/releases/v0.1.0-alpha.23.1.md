# Gyro v0.1.0-alpha.23.1

This follow-up release makes active chats easier to steer, improves local image
attachments, preserves the real order of provider activity, and discovers
signed updates sooner without disturbing an update already in progress.

## What changed

- Follow-up messages can now be queued while a response is running. The queue
  is FIFO, visible above the composer, and keeps Stop separate from Send.
  Waiting messages can be removed, while a message already being dispatched is
  protected from accidental removal.
- Queue execution is bounded to eight messages per chat and 24 across the app.
  Each item snapshots its attachments, workspace, provider, model, reasoning,
  approval policy, mode, goal, and plan. A workspace or provider preflight
  failure leaves the message queued instead of silently discarding it.
- PNG, JPEG, and WebP files can be dropped anywhere on the Chat surface. Private
  attachment previews use Tauri's asset protocol scoped only to Gyro's session
  attachment directory. Mixed drops retain valid images, report rejected
  items, and use a safe fallback when a preview cannot load.
- Transcript turns and events are ordered by their recorded timestamps.
  Repeated command, search, and tool activity is compacted only when adjacent;
  a live file row can transition from Editing to Edited or Edit failed without
  hiding separate operations elsewhere in the audit trail.
- Goal and Plan mode are mutually exclusive in the composer and at dispatch.
  Entering Plan mode clears the active goal, while setting a goal first switches
  to Normal mode and aborts safely if that transition cannot be persisted.
- Automatic update discovery is due every 30 minutes while Gyro stays open.
  A lightweight one-minute poll handles timer drift and invalid stored
  timestamps, while automatic checks pause during available, downloading,
  ready, and installing states so an update handle is never replaced
  underneath the user.
- Composer and provider menus now stay above surrounding Chat chrome. Image
  previews, light-theme context pills and session actions, user-message
  attachment ordering, queue rows, and hidden-sidebar titlebar clearance were
  polished across supported desktop sizes.

## Behavior changes and known limits

- Queued follow-ups are intentionally limited to the current app run and send
  only while their chat is selected. Switching chats pauses that queue; return
  to the chat to continue. This avoids invisible background provider calls and
  prevents automatic replay after a restart or update installation.
- A queue that reaches its per-chat or app-wide limit must be allowed to drain,
  or have a waiting item removed, before another follow-up can be added.
- Native image drops accept PNG, JPEG, and WebP only. Existing attachment magic,
  hash, path, size, count, and private-storage quota validation still applies.
- No database or session-schema migration is required.

## Upgrade and rollback

Update from Gyro or replace the existing app with the new DMG. CLI users can
install the matching archive; a generated Formula is included as a release
asset for manual installation or publishing to a tap.

Before updating or quitting, review any waiting follow-up messages because the
queue is intentionally not persisted between app runs. Existing sessions,
attachments, approvals, and Alpha 23 mutation-recovery state remain compatible
with Alpha 23.1. To roll back, back up
`~/Library/Application Support/Gyro/` and reinstall `v0.1.0-alpha.23`.

## Alpha notice

This is a GitHub-built public macOS alpha for Apple Silicon and Intel. Updater
archives are signed, but app bundles and DMGs are not Apple-signed or notarized.
