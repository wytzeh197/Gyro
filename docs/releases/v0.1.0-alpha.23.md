# Gyro v0.1.0-alpha.23

This release hardens Gyro's local backend for faster long-running chats, safer
provider execution, and more reliable recovery after interruption.

## What changed

- Bounded provider streaming, workspace scans, Git inspection, terminal output,
  IPC, and IDE protocols, with concurrency limits that keep resource growth
  bounded and reduce freeze risk during long-running work.
- Crash-resistant local state through atomic configuration writes, locked
  session logs with partial-tail repair, idempotent turn persistence,
  coordinated session cleanup, and recoverable mutation journals.
- Stronger execution security: Plan mode is enforced as read-only; provider,
  model, workspace, and approval policy are rebound to persisted native state
  at dispatch; approvals are bound to the active run and expire; previews,
  files, and attachments are validated and size-limited.
- Task and test launches are revalidated against native discovery, while
  language-server and debug-adapter commands use native allowlists. Unexpected
  renderer-supplied command substitutions are rejected.
- Cleaner shutdown and reaping for terminal and provider process trees, with
  direct-child cleanup for language servers and debug adapters.
- Updated Rust dependencies, including security fixes in the `plist` and
  `quick-xml` chain and the `portable-pty` 0.9 runtime update.
- Background chats retain session-scoped updates and show a working indicator;
  completed thought activity can be collapsed, and user messages now expose
  time, copy, and reuse actions on hover.

## Behavior changes and known limits

- When a saved Codex or Claude cursor is stale, Gyro clears it and asks for an
  explicit retry instead of automatically replaying the turn.
- Built-in terminal profiles now start in the user's home directory. Explicit
  commands and workspace terminal layouts continue to use their selected
  working directory.
- Remote OIDC login fails closed until issuer-backed JWKS signature verification
  is implemented. Default local-device access is unaffected.
- Excessively large attachments, outputs, protocol messages, or concurrent runs
  are rejected, truncated, or stopped with an explicit status, depending on the
  operation, instead of consuming unbounded resources.

## Upgrade and rollback

No database or session-schema migration is required. Update from Gyro or replace
the existing app with the new DMG. CLI users can install the matching archive;
a generated Formula is included as a release asset for manual installation or
publishing to a tap.

Gyro now refuses symlinked private-data directories and tightens their
permissions. If `~/Library/Application Support/Gyro/` or its private data files
are symlinks, restore them as regular local directories and files before
upgrading.

Before rolling back, resolve or reject pending file approvals. If Gyro crashed
during an approved mutation, relaunch alpha.23 once and let recovery finish
before quitting; alpha.22 does not understand alpha.23's in-flight mutation
state. Then back up `~/Library/Application Support/Gyro/` and reinstall
`v0.1.0-alpha.22`.

## Alpha notice

This is a GitHub-built public macOS alpha for Apple Silicon and Intel. Updater
archives are signed, but app bundles and DMGs are not Apple-signed or notarized.
