# Gyro v0.1.0-alpha.29.2

Gyro Alpha 29.2 reorganizes Settings around clearer product areas and makes
goal editing more reliable across chat modes.

## Download and install

Gyro requires macOS 14+ and is not Apple-signed or notarized. The application
uses a complete ad-hoc signature, so the first launch still requires Apple's
one-time **Open Anyway** approval.

Use the [Gyro download site](https://usegyro.io/) to select the correct build,
then follow the
[macOS installation guide](https://github.com/wytzeh197/Gyro/blob/main/docs/install-macos.md).
Do not disable Gatekeeper or remove quarantine globally.

- [Apple Silicon DMG](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.29.2/Gyro_0.1.0-alpha.29.2_aarch64.dmg)
- [Intel DMG](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.29.2/Gyro_0.1.0-alpha.29.2_x64.dmg)
- [SHA256SUMS](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.29.2/SHA256SUMS)

## What changed

- Settings now groups preferences, AI and agent controls, workspace controls,
  and system information into clearer sections.
- Editor, search, language-server, contribution, and workspace keybinding
  controls now live in the main Settings surface instead of a partial Explorer
  view.
- Settings navigation returns to the correct Sessions or Workspace destination
  and routes legacy workspace-settings state safely.
- Creating or editing a goal waits for the chat-mode change to succeed before
  clearing the goal composer, preventing lost text when a transition fails.
- Goal drafts are isolated from ordinary chat drafts, so opening, cancelling,
  or submitting the goal composer no longer disturbs a message in progress.
- The new-session menu presents CLI creation more directly for the active
  project.

## Behavior changes and known limits

- Workspace settings remain scoped to the selected project and its capability
  policy.
- Local language servers and contributions are detected and managed, but Gyro
  does not install them automatically.
- Gyro remains an unsigned public Alpha. Managed Macs may prohibit **Open
  Anyway**, and Gyro does not bypass device policy.
- Apple Silicon and Intel remain separate downloads; there is no universal DMG.
- Homebrew installs the `gyro` CLI only. Gyro.app remains a direct DMG install.
- Existing Alpha 29 sessions, projects, settings, goals, and editor state remain
  compatible; no destructive data migration is required.

## Upgrade and rollback

Alpha 29 data remains compatible with Alpha 29.2. Update from Gyro or replace
the existing application with the matching Alpha 29.2 DMG.

Before changing versions, quit Gyro and back up
`~/Library/Application Support/Gyro/`. To roll back, reinstall
[`v0.1.0-alpha.29.1`](https://github.com/wytzeh197/Gyro/releases/tag/v0.1.0-alpha.29.1).

## CLI

This release does not change CLI command behavior. After the release Formula
reaches the tap:

```bash
brew tap wytzeh197/tap
brew trust --formula wytzeh197/tap/gyro
brew install gyro
```

Direct archives are available for
[Apple Silicon](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.29.2/gyro-cli-0.1.0-alpha.29.2-aarch64-apple-darwin.tar.gz)
and
[Intel](https://github.com/wytzeh197/Gyro/releases/download/v0.1.0-alpha.29.2/gyro-cli-0.1.0-alpha.29.2-x86_64-apple-darwin.tar.gz).

## Alpha notice

This is a GitHub-built public macOS Alpha for Apple Silicon and Intel. Updater
archives are signed with Gyro's updater key, while application bundles use
ad-hoc code signing and DMGs are not Apple-signed or notarized.
